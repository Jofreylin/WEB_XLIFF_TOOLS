Unify XLF - XLIFF

How does it works?

This functionality allows you to upload and process two XLIFF files. The first file, referred to as the "Source File", should contain both <source> and <target> elements, while the second file, referred to as the "Target File", should contain <source> elements but potentially empty <target> elements. The script reads both files and compares the <source> elements as complete objects. If a match is found between the <source> elements of the Source File and the Target File, the corresponding <target> element from the Source File is copied into the Target File. This process ensures that the translations (targets) are transferred accurately from one file to another based on the matching sources. The resulting content of the Target File, with the updated <target> elements, is displayed for review. Additionally, there are options to clear the input fields and process the files again if needed.
